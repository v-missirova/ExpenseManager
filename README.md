# Expense Manager

Застосунок для обліку фінансів (гаманці та транзакції).

## Логіка та взаємодія класів (Архітектура)

Рішення розділене на 5 логічних проєктів:

1. **DBModels (Шар даних):** 
   - `WalletDBModel` та `TransactionDBModel` відповідають виключно за зберігання стану.

2. **UIModels (DTO):** 
   - `WalletUIModel` та `TransactionUIModel`. 

3. **Services:**
   - `FakeStorage`: Ізольоване (`internal`) штучне сховище з початковими  даними.
   - `StorageService`: Виступає посередником. Отримує "сирі" дані з бази і перетворює (mapping) їх на готові UI-моделі для відображення.

4. **ConsoleApp (Клієнтський шар):**
   - Точка входу (`Program.cs`).
     
5. **Common**
   - 'Currency`, `Category` - enums для зберігання даних про типи валют/категорій транзакції.

## Команди в консолі
Усі команди є нечутливими до регістру (case-insensitive):
- Ввід назви гаманця (напр. `mono_uah`) — перехід до перегляду повної інформації про транзакції вибраного гаманця.
- `back` — повернення на головний екран (список гаманців).
- `refresh` — скидання кешу та повторне завантаження сутностей зі сховища.
- `exit` — безпечне завершення роботи застосунку.
