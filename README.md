# Expense Manager

Застосунок для обліку фінансів (гаманці та транзакції).

## Логіка та взаємодія класів (Архітектура)

Рішення розділене на 7 проєктів:

1. **ExpenseManager.Common** Спільні типи даних: переліки (`enums`)

2. **ExpenseManager.DBModels** Моделі бази даних . Описують структуру таблиць SQLite (`WalletDBModel`, `TransactionDBModel`).

3. **ExpenseManager.DTOModels** Об'єкти перенесення даних (Data Transfer Objects). 

4. **ExpenseManager.Repositories** Рівень абстракції над даними. Містить інтерфейси та класи для виконання CRUD-операцій.

5. **ExpenseManager.Services** Шар бізнес-логіки. Виконує обробку фінансових операцій, валідацію та перетворення між DB-моделями та DTO.

6. **ExpenseManager.Storage** Реалізація фізичного збереження даних. Відповідає за налаштування SQLite-з'єднання, ініціалізацію бази даних та вигрузку початкових даних.

7. **MauiApp1** Клієнтський рівень (UI). Реалізований на платформі .NET MAUI з використанням патерну **MVVM** (Views та ViewModels).


## Функції

Поточний баланс, додавання/видалення гаманців, додавання/видалення транзакцій в гаманці.

## Команди в консолі версії (Legacy)

Усі команди є нечутливими до регістру (case-insensitive):
- Ввід назви гаманця (напр. `mono_uah`) — перехід до перегляду повної інформації про транзакції вибраного гаманця.
- `back` — повернення на головний екран (список гаманців).
- `refresh` — скидання кешу та повторне завантаження сутностей зі сховища.
- `exit` — безпечне завершення роботи застосунку.
